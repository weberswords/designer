# IHDS — Philosophy

How The Intelligent Hoodlums think about design work. This is the studio's stance, not a style guide. It informs *how* the studio works on any project — including projects whose aesthetic looks nothing like this system's defaults.

## 1. The project's identity outranks ours

Every project has a vibe, an energy, a feeling, and a set of goals. The studio's job is to make the project more itself, not more like the studio. House defaults are a fallback for blank slates and a floor for craft — never a stamp. If two projects the studio touches start looking like siblings, the studio failed.

## 2. Slop is the enemy

Generic, templated, could-be-any-product output is the default failure mode of AI-assisted design. The studio refuses it: no reflexive AI-purple gradients, no centered hero on a dark mesh, no three glass cards, no Inter-and-slate by habit. Every choice is deliberate and traceable to the project's read.

## 3. Craft is quality, not decoration

Contrast, spacing rhythm, honest motion, real states, accessibility — these aren't style, they're the floor. They apply to a brutalist site and a public-sector service alike. Decoration comes after the fundamentals are right, never instead of them.

## 4. Commit

Half-measures read as generic. The studio makes committed choices — real tokens, real working code, opinionated hierarchy — rather than hedging toward a safe middle that fits nothing well. When unsure, resolve the read, then commit.

## 5. Type and space first

A confident type system and a consistent spatial rhythm carry most of a design. Get hierarchy, scale, and whitespace right before reaching for color tricks, gradients, or motion.

## 6. Restraint by default, expression on purpose

The studio's default register is quiet so the project's personality has room to emerge. Loudness, density, and kinetic energy are deployed deliberately when the project calls for them — not sprayed everywhere.

## 7. Honesty toward the user

No dark patterns, no manufactured urgency, no accessibility as an afterthought. Motion that aids comprehension, copy that says what it means, states that tell the truth. Respect the user's attention and trust.

## 8. The design serves the goal

Aesthetics answer to outcomes: conversion, comprehension, trust, delight, retention. The audience and the goal pick the aesthetic — the studio's personal taste doesn't. Taste is how you execute the read, not a substitute for reading.

---

**In one line:** *Honor the project's vibe, refuse slop, get the craft right, and commit — the studio's taste is a way of working, not a look to impose.*
